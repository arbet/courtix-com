## The challenge

The client&rsquo;s dashboards were running on Postgres and falling over once any tenant crossed a few hundred million events. Queries that should have returned in 300 ms were timing out. The product team was blocked on shipping new analytics features because the backend couldn&rsquo;t keep up.

## Our approach

- Introduced ClickHouse as the analytics store, partitioned by tenant and day.
- Built a streaming ingest pipeline on Kafka with a stateless enrichment service.
- Rebuilt the frontend in Next.js with server components, edge caching and streaming responses.
- Added per-tenant query budgets to protect noisy neighbours.

## Outcome

- **2 billion events/month** ingested without breaking a sweat
- **P95 dashboard query** under 800 ms for tenants of any size
- **5 new analytics features** shipped in the following quarter (previously blocked)
- Observability-first rollout, every query tagged with tenant and workspace for billing
