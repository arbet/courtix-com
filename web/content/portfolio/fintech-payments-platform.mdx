## The challenge

A Series B fintech needed to replace an ageing payments orchestration layer that couldn&rsquo;t keep up with transaction volume. Latency spikes during reconciliation were breaching SLAs with their largest merchants, and scaling the existing Kubernetes footprint was pushing infra costs into six figures per month.

They wanted to rebuild on a modern, globally distributed stack — without taking the product offline or replaying a year of tests.

## Our approach

- Mapped every payment flow and produced a threat model before writing code.
- Modelled the new API as a set of Cloudflare Workers fronted by a regional Postgres primary with read replicas.
- Introduced idempotency keys, outbox-pattern event publishing and at-least-once reconciliation workers.
- Shadow-ran the new system against production traffic for six weeks before cutover.

## Architecture highlights

- Cloudflare Workers for the edge API, hitting PostgreSQL via Hyperdrive
- Redis for rate limiting and short-lived idempotency state
- Infrastructure-as-code with environment parity across dev, staging and prod
- Structured logs and traces shipped to the client&rsquo;s existing observability stack

## Outcome

- **3× increase** in sustained transactions per second
- **40% reduction** in monthly cloud spend
- **P95 API latency** dropped from 480 ms to 110 ms globally
- Zero downtime during the rolling cutover
